我的TestNG学习笔记：
  • TestNG官网
    1. Hard dependencies
      • 依赖的所有方法全部都成功执行后才会执行，如果有至少一个依赖失败，当前类或方法会被在报告中标记为SKIP。
      • alwaysRun=false
      • 支持dependsOnMethods和dependsOnGroups, 如下例： 
        	@Test
	public void serverStartedOk() {}

	@Test(dependsOnMethods = { "serverStartedOk" })
	public void method1() {}
    2. Soft dependencies
      • 不管依赖的方法是否都成功执行，始终会执行当前类或方法。
      • alwaysRun=true
    3. Dependencies in XML
      • Groups中有3个子菜单：define/run/dependencies。
      • define和dependencies放在一起的时候，dependencies不生效，也就是两个最好不要同时用。
      • define或dependencies最好和run一起使用，且run放在最上面，否则在eclipse中会提示错误，无法运行，原因未知。
      • 参照下例: 
        	<!DOCTYPE suite SYSTEM "http://testng.org/testng-1.0.dtd" >
	<suite name="testNgSuite" >
		<test name="Test dependencies" >
			<groups>
				<run>
					<include name="group7" />
				</run>
				<dependencies>
					<group name="group5" depends-on="group2 group4" />
					<group name="group7" depends-on="group5 group6" />
				</dependencies>
			</groups>
			
			<classes>
				<class name="testng.groups.MyTestB" />
			</classes>
		</test>
	</suite>
